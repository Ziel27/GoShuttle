GoShuttle Application Documentation
Version: 1.0.0
Audience: Non-technical users, operators, and stakeholders

============================================================
1) What GoShuttle Is
============================================================
GoShuttle is a transport coordination system with three connected parts:

1. Passenger Mobile App
   - For riders to request pickups, track updates, view announcements, and check ride history.

2. Driver Mobile App Experience
   - For drivers to start/end shifts, manage boarding, and handle active ride requests.

3. Admin Web Dashboard
   - For operations/admin teams to manage communities, shuttles, users, announcements, remittances, and analytics.

In simple terms, GoShuttle helps communities move people safely and efficiently while keeping trip and payment records organized.

============================================================
2) Who Uses the App
============================================================
Passenger
- Requests rides and sees service updates.
- Views recent rides and fare history.
- Manages home destination and notification preferences.

Driver
- Starts and ends duty shifts.
- Handles passenger boarding and dispatch-related tasks.
- Resolves pending ride issues before closing shift.

Admin
- Oversees daily operations in the web dashboard.
- Publishes service announcements.
- Reviews fleet activity, remittances, and performance analytics.

============================================================
3) How the System Works (High Level)
============================================================
Step 1: User signs in
- Users log in with email and password.
- Password reset is available if needed.

Step 2: Role-based experience loads
- The app automatically shows features based on role (Passenger or Driver).
- Admins use a dedicated web dashboard.

Step 3: Real-time operations
- Ride and announcement updates are delivered live.
- Users can refresh screens manually when needed.

Step 4: Operations and records
- Driver shifts, ride events, and remittance-related actions are tracked.
- Admin dashboard summarizes trips, passenger counts, and revenue trends.

============================================================
4) Passenger Journey (What Riders Usually Do)
============================================================
Home / Main screen
- Opens the rider experience and operational status view.
- Shows ride-related context and available actions.

Rides tab
- Shows recent rides with:
  - Shuttle details
  - Boarding/request time
  - Fare paid
  - Pickup and destination labels
- Includes filters (Last 7 days, Last 30 days, All).

Fleet/Ops tab
- Provides rider fleet/operations view based on role context.

Announcements tab
- Displays service updates from community admins.
- Announcement types include Info, Warning, and Critical.
- New announcements can appear in near real-time.

Settings tab
- Manage account, notifications, and app preferences.
- Set Home Address + GPS for easier pickup and destination handling.
- Option to enable/disable precise pickup sharing.
- Theme preferences (System, Light, Dark).

============================================================
5) Driver Journey (What Drivers Usually Do)
============================================================
Start shift
- Driver starts a shift from Settings.
- Driver status changes to active/driving.

During shift
- Handles passenger boarding flow.
- Responds to ride request and dispatch events.
- Uses notifications for important actions.

End shift
- Driver must resolve pending ride requests before shift closure.
- System prevents incomplete shift closure when unresolved requests exist.
- Shift summary can include passenger count and revenue totals.

Logout safety
- If a driver is still on shift, the app prompts them to end shift first.
- This protects record accuracy for trips and remittances.

============================================================
6) Admin Dashboard (Web)
============================================================
Main operational areas include:
- Overview: key performance cards and daily trends
- Communities: service area and destination management
- Users: user and role oversight
- Shuttles: fleet management
- Remittances: submission review and verification
- Analytics: operational and performance insights
- Announcements: create and publish updates to users

Admins can monitor the health of operations and quickly communicate changes to passengers and drivers.

============================================================
7) Notifications and Service Updates
============================================================
Users can control:
- Push Alerts
- Service Updates
- Quiet Mode

Behavior notes:
- Critical operational messages can still be prioritized.
- Announcement and alert behavior follows user preference settings.

============================================================
8) Privacy and Data Handling (User-Level View)
============================================================
GoShuttle uses account and location information to provide routing and dispatch functionality.

Passenger controls include:
- Home destination label updates
- Optional precise pickup behavior
- Notification preferences

The app is designed so users can adjust these preferences from Settings.

============================================================
9) Common Questions
============================================================
Q: Why do I see different screens from another user?
A: Features depend on your role (Passenger, Driver, or Admin).

Q: Why can’t a driver log out immediately?
A: If a shift is still active, the app asks the driver to close it first to keep records accurate.

Q: Where do service advisories appear?
A: In the Announcements tab, and optionally via notifications depending on your settings.

Q: Can I change my home destination?
A: Yes. In Settings, update the home address and save GPS location.

============================================================
10) Summary
============================================================
GoShuttle is built to make community shuttle operations organized, traceable, and user-friendly:
- Passengers request and track rides
- Drivers run structured shift workflows
- Admins manage operations and monitor outcomes

The result is clearer dispatch coordination, better communication, and more reliable transport service delivery.
